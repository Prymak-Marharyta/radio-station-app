**Radio Station Management System**
-----------
**Автори**

Я виконую цей проєкт самостійно та поєдную всі ролі:
- Team Lead
- Developer 1
- Developer 2
-----------

**Опис проєкту**

Це GUI-додаток, розроблений на JavaFX, який працює з базою даних через JDBC.
Мета застосунку - забезпечити зручну роботу з даними радіостанції:
- перегляд записів
- додавання нових
- редагування
- видалення
- пошук
-----------

**Предметна область**

Інформаційна система радіостанції зберігає дані про:
- артистів
- композиції
- ефіри

**Основні запити:**
- треки виконавця за минулий місяць
- треки виконавця за сьогодні
- виконавець з найбільшим ефірним часом за тиждень

**Бізнес-правило:**
- Кожна композиція має лише одного виконавця
-----------

**Технології**
---
- Java

- GitHub

-----------

**Робота з Git**
---

**Структура гілок:**
- main - стабільна версія
- develop - розробка
- feature/* - новий функціонал

**Правила комітів:**
- feat: додано нову функцію
- fix: виправлення помилки
- update: зміни в коді
- docs: документація

-----------

**Правила злиття (Merge)**
---
- Усі зміни виконуються через Pull Request
- Перед злиттям перевіряється працездатність
- Злиття виконується в гілку develop, потім у main
-----------

**План роботи**
---

**Частина 1:**
- створення репозиторію
- опис README
- визначення структури БД

**Частина 2:**
- підключення до БД через JDBC
- створення DAO класів
- реалізація CRUD операцій

**Частина 3:**
- створення GUI (JavaFX)
- підключення GUI до БД
- реалізація пошуку
-----------

**Функціонал застосунку**
---
- додати запис
- видалити запис
- редагувати запис
- переглянути всі записи
- пошук по назві/артисту
-----------

**Особливості**
---
- використовується H2 (не потребує встановлення)
- простий та зрозумілий інтерфейс
- дотримано бізнес-логіки (1 артист = 1 трек)
